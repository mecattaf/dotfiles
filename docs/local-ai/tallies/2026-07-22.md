# Local-AI tally — 2026-07-22 anchor

Cutoff: 2026-07-22, Europe/Paris. This is the first full-roster normalization,
following the July repository-delta dry run. It establishes the baseline that
future monthly reports will update.

## Gate and scope

- `services.local-models.downloadAllModels = false` remains unchanged.
- No model blob was downloaded, copied, deleted, built, or added to a Nix
  closure. Hugging Face access in this pass was API/model-card metadata only.
- This pass declares metadata and documentation. It does not add the future

## Decisions from the full pass

### General and coding

Qwen 3.6 35B MXFP4 stays the daily default because the exact live HF object is
the object in the `tesla_agent` stable manifest and has a matched Strix Halo
quality/speed witness. Qwen3-Coder-Next is the purpose-built coding opinion.
Qwopus remains a separate row—not “Qwen 27B”—because its fine-tune, Q5_K_M
artifact, and 42/148 BigCodeBench-Hard run form a distinct tuple.

Gemma 4 supplies a different training family to the coding pool. On 2026-07-17,
Google replaced the QAT GGUF with corrected-vocabulary revisions. The prior
published Strix row used OID
`4c856523d61d77922dbc0b26753a6bf6208e5d69d80db0c04dcd776832d054c5`;
the selected current file uses
`3eca3b8f6d7baf218a7dd6bba5fb59a56ee25fe2d567b6f5f589b4f697eca51d`.
The new file is canonical metadata but `unverified` evidence until a matched
local run is recorded.

### SOTA / DS4

DeepSeek V4 Flash remains Tom's SOTA lane. The selected identity is the full
164,633,502,592-byte Q4 imatrix artifact plus the 3,807,602,400-byte MTP head,
not the single-node IQ2 or hybrid capacity variants. The actual Nix runtime is
the pinned `ejpir/ds4-hip` package, not an old container and not a direct
unmanaged port.

The current catalog schema cannot yet express the two different process roles
(`coordinator 0:21`, `worker 22:output`) as one llama-swap deployment. The row
therefore records final identity and intent while the closed gate prevents a
false single-node launch. Fleet orchestration is a required deployment followup.

### Uncensored pool

The pool deliberately varies both model family and refusal-removal route:

1. Qwen 3.6 processed with a Heretic MPOA/SOMA-style route;
2. SuperGemma 4's tuned uncensored line, with an exact 66.07 tok/s Vulkan
   throughput witness on Strix Halo;
3. GLM 4.7 Flash Aggressive, a non-Heretic HauhauCS route.

The previously considered Hy3-Heretic GGUF was not selected: its model card
requires an unmerged DS4 Hy3 branch/PR rather than the packaged inference
runtime. These models remain manual hypothesis generators and are excluded from
automatic routing.

### OCR and embeddings

The matched academic-rag result wins over novelty: Qwen3-VL 8B Q8 remains the
primary drainer, 32B Q4 is the conditional table/math second pass, and Qwen3
Embedding 8B remains the companion embedder. The executable scripts point to
Q5_0. Older ledger strings that called the embedder Q8_0 were stale labels, so
the catalog resolves the identity to Q5_0 rather than silently triggering a
full re-embedding.

### Speech

The current `asr-rs` dual model contract remains the streaming choice: the 120M
end-of-utterance model for preview and Parakeet TDT 0.6B v2 for finalization,
both CPU-side. VibeVoice-ASR-HF is selected for long-call transcription plus
speaker attribution; it is not incorrectly attributed to Kyuz0's TTS-only
voice toolbox. VibeVoice Large remains the TTS choice because a Strix-specific
ROCm runtime exists, with its community-mirror/retracted-upstream risk stated.

## Mechanical normalization used this month

This is the repeated work that should become the deterministic part of the
monthly tally:

1. Resolve a benchmark display row to its structured run record and exact
   profile/file path. Never promote from a display label alone.
2. Resolve the canonical HF repository and immutable revision through the HF
   metadata API; read the model card for runtime/fine-tune lineage.
3. Record every required file, exact byte count, LFS SHA-256/OID, and convert
   the OID to Nix SRI. This uses metadata only; do not request blob endpoints.
4. Pin the inference implementation actually packaged by the dotfiles. A model
   plus a different fork is a different tuple.
5. Attach the benchmark source commit and run ID only when the artifact mapping
   is exact. Otherwise set evidence to `unverified` and explain the gap.
6. Reconcile by stable deployment ID: add, supersede, retire, or retain. A
   no-delta champion remains in the roster.
7. Evaluate the typed catalog and render the report with the download gate
   false. The monthly mechanism may propose a patch; it may never lift the gate.
